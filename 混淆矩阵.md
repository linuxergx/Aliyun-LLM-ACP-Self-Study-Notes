### 😅1. 什么是混淆矩阵？（以及为什么要吐槽它） 

 在机器学习和 NLP（大模型）领域，**混淆矩阵（Confusion Matrix）**是衡量模型“靠不靠谱”的唯一真理。简单来说，它就是一张记录模型“==预测结果==”与“==真实情况==”对碰的得分表。  混淆矩阵是监督学习中用来“打分”的工具。 打分之后就可以根据各项分值来建立一套指标，也就是后面要说的精确率 (Precision)、召回率 (Recall)、F1 Score。

**监督学习**：

* 监督俩字可以把它理解成你有一套标准答案或者固定标准，而监督学习就是你通过做题之后对比标准答案的过程。
* 再举一个例子，比如你有一套酸碱试纸，你通过检测试纸与比色卡之间差异的过程。

**预测结果(大模型回答的）与真实情况( ground_truth )的展示**

```json
'预测结果': [
        '张伟是教研部负责大模型的同事。' # 这个回答会大模型拆分成 “张伟是教研部的”，“张伟是负责大模型的”
    ],
    '真实情况':[
        '张伟是教研部的成员',
        '张伟负责大数据方向的研发'
    ]

# 在评分过程中会先将answer与ground_truth进行原子化拆分，分析各项中的信息点进行分类。也就是说只有answer与ground_truth中起码有2个信息点的情况才能进行分类，也只有这种前置条件才会存在模型回答时漏答的现象（FN）。
#拆分与打分过程使用到了大模型（代码中的llm=Tongyi(model_name="qwen-plus")）
```



####  槽点：绕晕人的“真阴假阳” 

 传统的教材总喜欢说：真阳性（TP） True Positive 、假阳性（FP） False Positive 、真阴性（TN） True Negative 、假阴性（FN） False Negative 。 

![img](https://i-blog.csdnimg.cn/blog_migrate/d7a4b71d1f2393e35df6988fd990c2d9.jpeg)

**吐槽时间：** 正常人看到这四个词，大脑第一反应绝对是“我是不是阳了？”这种医学术语极度反直觉，尤其是“假阴性”和“假阳性”，绕两圈你就忘了谁是谁了 。

### 😉2. 建立理科生的逻辑映射：P 是动词，N 是沉默 

 我们要打破死记硬背，必须建立一套**“动作映射”**。把 **P (Positive)** 看作“动词（模型的动作）”，把 **N (Negative)** 看作“沉默（不存在）”。 

#### 核心逻辑公式： 

- **P (Positive) = 动作/存在**：模型“开口说话”了，只要说话必定存在说对与说错，TP说对了（True）FP说错了（False），模型开始胡说八道了,出现了幻觉。
- **N (Negative) = 沉默/缺失**：模型“闭嘴没说”，没说也存在两种情况，漏说了FN，本来就没有的也确实没说TN（好比密封线内不准答题，你也确实没有在里面答题，因为答题了也不给分，所以一般在计算精确度和召回率的时候也不会把TN算进去，类似的情况有太多太多，对于那两个指标来讲它也就不具备统计意义）。

| 模型动作     | 事实(GT)   | 我的总结        | 简单逻辑            |
| :----------- | :--------- | :-------------- | :------------------ |
| **说了 (P)** | **有 (T)** | **说对了 (TP)** | 该说且说对了        |
| **说了 (P)** | **无 (F)** | **放屁 (FP)**   | **胡说八道 (幻觉)** |
| **没说 (N)** | **有 (T)** | **漏了 (FN)**   | **话说一半 (漏答)** |
| **没说 (N)** | **无 (F)** | **真沉默 (TN)** | 该闭嘴时闭嘴        |





### 😁3. 两个关键维度的博弈：P 与 R 

 理解了上面的动作，你就知道为什么要搞出 **精确率 (Precision)** 和 **召回率 (Recall)** 了： 

#### **精确率 (Precision)：**

考核大模型说的话里面有多少是真话 

* 公式：$P = \frac{TP}{TP + FP} = \frac{说对的话}{说对的话+胡说的内容}$

* 目的：  看大模型说出来的每句话里，有多少不是在“放屁”。 
*  **应用场景**：比如医疗助手，我可以忍受你话少，但你绝对不能胡编乱造（FP 必须极低） 

#### 召回率 (Recall)：

本该大模型回答的内容它说了多少 ，类比： 金矿的纯度

* **公式**：$R = \frac{TP}{TP + FN} = \frac{说对的话}{说对的话+漏掉的话}$

* **目的**： 标准答案里有多少是大模型说对的，像是做试卷，满分100实际写对的有多少。
*  **应用场景**：比如法律检索，漏掉一个法条可能就败诉了，所以必须挖得全（FN 必须极低）。 

### 🎇4. 终极评分：为什么 F1 必须是“调和平均”？ 

> 在此之前我们要先知道，我们要的模型一定是一个精确率 (Precision)和召回率 (Recall)分值都很高的模型，根据这两个分值我们才会有基于精确率 (Precision)和召回率 (Recall)的评价标准。

 如果 P 和 R 同样重要，为什么不直接取 (P+R)/2？ 

#### 算术平均的漏洞 

 假设一个模型极其懒惰，只说了一句绝对正确的真话，剩下 99 个点全部漏掉。 它的 P=100%, R=1%。 

 **算术平均**：${\displaystyle\frac{100+1}{2}=50.5}$（居然及格了？不合理！） 



#### 调和平均

 在数学上，平衡两个率（Ratio）最公平的方式是取它们**倒数的算术平均**，再取倒数回来。 

* **目的**： 我们需要 **F1 Score** 来惩罚偏科生，它是一个只要有一个值为 0，结果就必须为 0的数学结构 

##### **推导过程：** 

1. 设 P 为精确率，R 为召回率。我们先求它们倒数的平均值：

    $M_{inverse} ={\displaystyle\frac{ \frac{1}{P}+\frac{1}{R}} {2}}$

2. 我们将这个“倒数平均值”再取一次倒数，就得到了 ：

    $F1=\frac{1}{\displaystyle{M_{inverse}}}=\frac{1}{\displaystyle \frac{\frac{1}{P} + \frac{1}{R}}{2}}$



 


 **分母通分**：${\displaystyle\frac{1}{P}+\frac{1}{R}} ={\displaystyle \frac{R+P}{P*R}}$

 

 **带回大分式** : $F1=\frac{1}{\displaystyle \frac{\frac{R+P}{P·R}}{2}}$

 **翻转分式** :  $F1=\frac{1}{\displaystyle \frac{R+P}{2PR}}$

 **最终形态**: $F1=2·{\displaystyle\frac{P·R}{P+R}}$

如果我们把这个 P（精确率）和 R（召回率）的定义代入 ，最终结果是：

```python
tp / (tp + 0.5 * (fp + fn))
```

最终的这个值代表了事实准确度， 你可以发现，FP（误报）和 FN（漏报）在 F1 中被摆在了**同等权重**的位置,都是50% ，调和平均的特性决定了，只要 P 和 R 有一个接近 0，F1 就会迅速趋近 0。它逼着模型既不准“FP”，也不准“漏答”。

### 5. 实战：大厂是如何打分的？（以阿里教案为例）

当你看到一个 **Answer Correctness** 的分数时，它其实是 LLM 和 Embedding 模型的“混合双打”：

```bash
在python里就是embedding 模型进行向量化（通过代码中的embeddings=DashScopeEmbeddings(model="text-embedding-v3")）来计算语义相似度。它不管你里面有几个事实，它只看整体语境、语气、用词像不像。即便你事实全错了，只要你说话的“味儿”对，这 0.25 的权重里你也能拿点分。
```



$$最终得分 = 0.75 \times 事实准确度(F1  Score) + 0.25 \times 语义相似度$$

* **75% 的权重给了事实**：逻辑底线是不放屁。
* **25% 的权重给了相似度**：这是考察表达得像不像人。
